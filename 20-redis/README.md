# Redis

Bitnami OCI chart, `architecture: standalone` by default. Installs once into the shared
`shared-infra` namespace.

## Purpose

- Shared store for Mosquitto's active/passive/passive HA (`10-mqtt-mosquitto`) — retained
  messages and the leader lock live here, not on any broker pod's local disk.
- Available as a general-purpose cache for any other infra/component that needs one.

## Connection details

- Host (cluster-internal): `redis-master.shared-infra.svc.cluster.local:6379`
- Password: auto-generated by the chart on first install, stored in the `redis` Secret
  (key `redis-password`) in `shared-infra`. Retrieve it with:
  ```powershell
  kubectl get secret redis -n shared-infra -o jsonpath='{.data.redis-password}' | base64 -d
  ```

## Switching to replication

If Mosquitto/EMQX HA ends up needing a real Sentinel-backed store instead of a single
Redis instance, change `Architecture` to `"replication"` in `Config.psd1` and re-run
`Install-Infra.ps1` — the chart handles the master/replica/sentinel topology itself.
